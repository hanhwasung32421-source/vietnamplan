# 수정 가이드
원본 파일은 그대로 두고, 수정/정제본은 별도 파일로 생성하는 방식을 권장합니다.
## 추천 작업 흐름
- 1) 목표 정의: 어떤 데이터를 최신화/정리/삭제/통합할지
- 2) 규칙 확정: 필드명 표준(예: 지역/업종/업소명/연락처/URL 등), 중복 판정 기준
- 3) 정제본 생성: `*_clean.json` 또는 `*.csv` 등 사람이 편집하기 쉬운 형태로
- 4) 검증: JSON 파싱 가능 여부, 중복/누락 통계 확인
- 5) 반영: 원본 교체가 필요하면 백업 후 교체

## 주의
- 텍스트에 연락처/아이디 등이 포함되어 있어 외부 공유 전 마스킹이 필요할 수 있습니다.
- 큰 JSON은 에디터에서 열기 무거울 수 있으니(특히 20MB+), 분할/CSV 변환을 권장합니다.
- 이번 폴더에는 연락처/아이디로 보이는 패턴이 많아, 공유용 파일은 `*_masked.csv`를 쓰는 것을 추천합니다.
